Music Store Analysis

This project focuses on analyzing customer behavior, invoice data, and music preferences using SQL queries on a structured database of a music store. The database contains interconnected tables that store transactional and customer-related data. Through a series of SQL queries, from basic data retrieval to advanced techniques like Common Table Expressions (CTEs) and recursive queries, the project aims to extract insights into sales trends, customer purchasing patterns, and music preferences. Key tables in the dataset include customer details, invoices, invoice lines, tracks, albums, artists, employees, and genres. By leveraging SQL, this project enables efficient data exploration, supporting business performance analysis, sales strategy optimization, and enhanced customer engagement.

Anomally detection on Mobile Banking Transaction

This project utilizes the Isolation Forest algorithm to detect anomalies in mobile banking transactions, enhancing fraud detection by identifying suspicious activities based on transaction amount deviations, frequency anomalies, and account behavior changes. Traditional rule-based systems often generate high false positives, making machine learning a more effective approach. The implementation involves data preprocessing, feature engineering, and model training using Python and PySpark. By isolating irregular transaction patterns, the model successfully reduces false positives and provides actionable insights for financial institutions, improving fraud prevention, transaction security, and risk management.
